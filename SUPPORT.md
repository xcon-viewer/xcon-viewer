# Support

This document explains where to ask for help with XCON Viewer.

## Questions and discussion

Use GitHub Discussions for:

- General questions.
- Ideas.
- Usage patterns.
- Syntax design discussion.
- Showcase posts.
- Integration questions.

Discussions:
https://github.com/xcon-viewer/xcon-viewer/discussions

## Bug reports

Use GitHub Issues for reproducible bugs.

Before opening an issue:

1. Search existing issues.
2. Prepare the smallest XCON or Markdown example that reproduces the problem.
3. Include package, version, environment, and expected behavior.

Issues:
https://github.com/xcon-viewer/xcon-viewer/issues

## Security vulnerabilities

Do not open public issues for suspected security vulnerabilities.

Read:
[SECURITY.md](./SECURITY.md)

## Documentation problems

Use the documentation issue template for:

- Broken links.
- Missing examples.
- Confusing wording.
- Outdated commands.
- Inconsistent syntax documentation.

## Feature requests

Use the feature request template for new features.

Good feature requests explain:

- The use case.
- Why current behavior is insufficient.
- Proposed behavior.
- Compatibility with XCON Viewer's safe-by-design model.

## Commercial or private support

This repository does not currently provide guaranteed private support through GitHub issues. If a commercial support channel is added later, document it here before advertising it elsewhere.
