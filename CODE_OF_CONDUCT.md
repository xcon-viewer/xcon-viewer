# XCON Viewer Code of Conduct

## Purpose

XCON Viewer is an open-source project for safely rendering structured UI documents. We want the project to be useful to maintainers, contributors, users, security researchers, and people evaluating XCON as a UI-document format.

This Code of Conduct defines the behavior expected in this repository, including issues, pull requests, reviews, discussions, release conversations, and related community spaces controlled by the project maintainers.

## Expected behavior

Participants are expected to:

- Be respectful, direct, and constructive.
- Assume good intent when possible, but correct technical problems clearly.
- Criticize ideas, code, and architecture without attacking people.
- Keep discussions relevant to XCON Viewer, XCON syntax, safe rendering, documentation, examples, integrations, and project maintenance.
- Help new contributors understand the repository before asking them to make large changes.
- Respect maintainers' time by using the issue and pull request templates.

## Unacceptable behavior

Unacceptable behavior includes:

- Harassment, threats, insults, or targeted personal attacks.
- Sexualized language or imagery.
- Discriminatory comments or exclusionary behavior.
- Publishing someone else's private information without permission.
- Trolling, repeated low-effort disruption, or intentionally derailing technical discussion.
- Pressuring maintainers to accept unsafe changes, especially changes that weaken the viewer-only security boundary.
- Publicly disclosing security vulnerabilities before the maintainers have had a reasonable opportunity to review and fix them.

## Security-sensitive discussions

Do not open public issues for suspected vulnerabilities.

Examples include:

- Script execution bypasses.
- Raw HTML injection paths.
- `javascript:` URL bypasses.
- CSS allowlist escapes.
- External resource loading bypasses.
- Parser or renderer behavior that could enable untrusted content execution.

Use the process in [SECURITY.md](./SECURITY.md) instead.

## Enforcement

Maintainers may take action when this Code of Conduct is violated. Actions may include:

- Asking for clarification or edits.
- Hiding, editing, or deleting comments.
- Closing issues or pull requests.
- Temporarily or permanently limiting participation.
- Reporting abusive behavior to GitHub when needed.

Enforcement decisions should be proportional, consistent, and focused on protecting the project and its contributors.

## Reporting conduct concerns

For public, non-sensitive concerns, contact the maintainers through GitHub Discussions.

For private concerns, use the private contact path listed in [SECURITY.md](./SECURITY.md). If the concern is itself a security vulnerability, follow the vulnerability reporting process there.
