The Problem

LLM responses are
stuck in plain text.

Your AI app answers brilliantly — but every response looks the same. A wall of markdown. No charts. No cards. No dashboards. The gap between what AI knows and what it can show is still wide open.

📄

Markdown only gets you so far

Headings and bullet points are not dashboards. Users expect their AI-powered tools to visualize data — not just describe it.

# Revenue\n- MRR: $124K...
🔓

Raw HTML from LLMs is dangerous

Letting LLMs generate HTML means XSS risks, inconsistent output, and no sandboxing. You cannot safely render untrusted HTML from AI responses.

<script>...</script> ← blocked
🏝️

Canvas tools don't embed

Claude Artifacts, ChatGPT Canvas — great products, but locked to their own platform. You cannot embed them in your own LLM app.

// embed not supported
The Solution

Markdown for UI.
Built for LLMs.

XCON is what Markdown is for text — but for UI. A structured, text-based format that LLMs generate naturally, renders safely, and embeds anywhere.

🧩

One npm install, any framework

React, Vue, vanilla JS, markdown-it, remark, Vite plugin — XCON drops into your existing stack without replacing anything.

🔒

Safe by design

Strict viewer-only boundary. No script execution, no event handlers, no raw HTML injection. Share UI from untrusted sources safely.

⚡

Streams as it generates

Best-effort partial rendering during streaming. The UI builds up in real time as the LLM outputs tokens — not after it finishes.

📐

Four syntaxes, one model

SKETCH for LLMs and Markdown. JSON for APIs. XML for tooling. TAGLESS for transport. All parse to the same object model.

Korea-first advantage

LLM can't read HWP.
XCON bridges the gap.

HWP is Korea's dominant document format — used daily by government, legal, and enterprise. But LLMs can't generate or parse it natively. XCON sits in the middle: Markdown + SKETCH that LLMs understand, with HWP/HWPX export for the Korean market. No other open-source tool does this.

Document pipeline
HWP / HWPX → Markdown
LLM generates SKETCH
XCON renders UI inline
Export → HWP / PDF / HTML
